JSX的本质

jsx -> babel -> React.createElement
实际上，jsx是React.createElement(component,props,...children)函数的语法糖，所有jsx最终都会被转换成React.createElement的函数调用.

两者渲染效果是相同的。

babel将jsx转换为React.createElement

可以到babel官网-Try it out查看。

React.createElement(type,config,children)的三个参数

type

  • 当前ReactElement的类型；
  • 如果是标签元素，那么就使用字符串表示，如'div'
  • 如果是组件元素，那么直接使用组件的名称，如'Spin'

    config

  • 所有jsx中的属性都在config中以对象的属性和值得形式存储
  • 例：<div className='spin'>loading</div>->React.createElement('div',{className:'spin'},'loading')

    children

  • 存放在标签中的内容
  • 如果只有一个子元素，正常传递，如果一个元素有多个子元素，将所有子元素按顺序排在config参数后

为什么React.createElement(type,config,children)可以接受大于3个的参数
